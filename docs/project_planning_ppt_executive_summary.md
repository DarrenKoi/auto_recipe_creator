## Slide 2. 왜 해야 하는가 (배경)

**Recipe 셋업은 숙련 엔지니어에게 전적으로 의존하는 반복 수작업 — 장비 증가에 인력이 비례해야 하는 구조적 한계**

| 현재 (As-Is) | 문제 |
|--------------|------|
| 엔지니어가 RCS GUI에서 수동으로 마우스/키보드 조작 | 반복 작업에 고급 인력 투입 |
| 측정 위치를 시행착오로 조정 (측정 → 실패 → 재조정 반복) | 작업 시간 과다, 품질 편차 |
| 노하우가 개인에 귀속 | 인력 이탈 시 지식 유실 |
| 장비 수 증가 = 인력 수 증가 | **스케일 불가 구조** |

---

## Slide 3. 무엇을 만드는가 (핵심 기술 & 아키텍처)

**VLM(화면 인식 AI) + RAG(전문가 지식 검색) + GUI 자동화를 결합한 Supervised Autonomy 시스템**

| 구성 요소 | 역할 | 핵심 포인트 |
|-----------|------|------------|
| **VLM** (Qwen3-VL) | 화면을 보고 다음 행동을 판단 | 사내 GPU 클러스터 활용, 외부 의존 없음 |
| **RAG** (CCTV 기반) | 유사 상황의 전문가 작업 이력을 검색하여 판단 보강 | Fine-tuning 없이 성공률 15~20%p 향상 |
| **GUI 자동화** | 판단 결과를 마우스/키보드로 실행 | 단일 액션 <200ms |
| **Supervised Autonomy** | AI 주도 + 엔지니어 언제든 개입 가능 | 안전한 단계적 자율화 |

```
  H200 GPU 클러스터          자동화 PC              엔지니어
 ┌──────────────┐      ┌──────────────────┐    ┌──────────────┐
 │ VLM 추론 서버 │◄────►│ 화면 캡처        │    │ 모니터링 UI   │
 │ CCTV 분석    │      │ AI 판단 (VLM+RAG)│◄──►│ 개입/채팅     │
 │ RAG 인덱스   │      │ 자동 입력 실행    │    │ 모드 전환     │
 └──────────────┘      │       ↓          │    └──────────────┘
                        │   RCS (장비 제어) │
                        └──────────────────┘
```

## Slide 4. 어떻게 진행하는가 (로드맵)

**5단계, 약 25주 — PoC 검증 후 파일럿을 거쳐 다중 장비 확장**

| Phase | 기간 | 핵심 목표 | 완료 기준 |
|-------|------|----------|----------|
| **1. 환경 구축** | 1~4주 | PC 세팅, 화면 캡처/입력 전달 검증, VLM 연동 | 기본 기능 PoC 통과 |
| **2. 핵심 기능** | 5~10주 | 상태 매칭, 측정 판단, 워크플로우 엔진, RAG | 핵심 테스트 케이스 통과 |
| **3. UI 개발** | 11~14주 | 엔지니어 모니터링/제어 화면, AI 채팅 | 엔지니어 인터페이스 완성 |
| **4. 파일럿** | 15~20주 | 실제 장비 1대 적용, 피드백 반영 | **성공률 70%↑, v1.0 릴리즈** |
| **5. 확장** | 21주~ | CCTV 기반 RAG 고도화, 다중 장비 배포 | **성공률 90%↑** |

## Slide 5. 얻는 것은 무엇인가 (기대 효과)

**수작업 50% 감축과 전문가 지식의 자산화를 동시에 달성**

| 정량적 효과 | 정성적 효과 |
|------------|------------|
| 작업 시간 **50% 이상 단축** | 전문가 노하우가 시스템에 축적 → **인력 이탈 리스크 해소** |
| 자동화 성공률 **90%** 달성 | 숙련도 무관하게 **일관된 작업 품질** 확보 |
| 사람 개입 **20% 이하**로 감소 | 1명이 다수 장비 모니터링 → **인력 확장 없는 장비 증설 가능** |
| 사이클 당 소요 **~3초** (수동 대비 대폭 단축) | CD-SEM, VeritySEM 이후 **다른 장비로 확장 가능한 플랫폼** |

## Slide 6. 리스크와 선행 조건

**주요 리스크는 사전 대응 설계 완료 — 프로젝트 시작을 위한 3가지 의사결정 필요**

| 주요 리스크 | 대응 (이미 설계됨) |
|------------|-------------------|
| VLM 정확도 부족 | RAG로 전문가 사례 보강 (+15~20%p), Fine-tuning 없이 해결 |
| VM 보안 제약 (외부 API 차단) | Host PC에서 이미지 기반 제어 — VM 내부 설치 불필요 |
| 엔지니어 수용성 | Supervised 모드로 시작, 강제 자동화 없음, 언제든 수동 전환 |

**의사결정 요청:**

| # | 요청 사항 | 필요 시점 |
|---|----------|----------|
| 1 | **자동화 전용 PC** 예산 승인 | Phase 1 착수 전 |
| 2 | **파일럿 대상 장비** 1대 지정 (CD-SEM 또는 VeritySEM) | Phase 4 전 |
| 3 | **CCTV 영상 접근 권한** 승인 | Phase 5 전 |
